Workflow Gitflow

Le Gitflow est un modèle de branchement strict conçu pour la gestion des releases de projets. Il définit un rôle très précis pour chaque branche.

La branche sacrée. Elle ne contient que du code prêt pour la production. On n'y commit jamais directement. Elle ne reçoit que des fusions provenant de Release ou Hotfix.

C'est la colonne vertébrale du projet. C'est ici que le code des fonctionnalités est rassemblé. Quand Develop est stable, on prépare une Release.

Utilisées pour développer de nouvelles features. Elles partent de Develop et reviennent dans Develop via une Merge Request.

Branche temporaire pour finaliser une version (v1.0). On y corrige les derniers bugs mineurs avant de fusionner vers Main et Develop.

La seule branche qui peut partir de Main. Utilisée pour réparer un bug critique en production immédiatement.
